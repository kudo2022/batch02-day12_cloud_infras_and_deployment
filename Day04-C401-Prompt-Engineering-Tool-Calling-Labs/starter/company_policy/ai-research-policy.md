## Research workflow

- Start with the narrowest reliable source for the user intent.
- Use `web_search` for current news and broad web discovery.
- Use `search_tweets` for social signals and discussion trends.
- Use `arxiv_search` for academic papers and recent preprints.
- Use `get_arxiv_paper_text` when the user provides a specific arXiv ID or URL and asks to inspect content.

## Verification requirements

- Do not present a claim as verified unless it is supported by a primary source or reputable reporting.
- For briefings, keep source URLs attached to each factual claim.
- If sources conflict, state the conflict instead of forcing one conclusion.

## Research output

- Executive-facing briefings must include business impact, risk, and recommended next step.
- Technical paper scans must include problem, method, result, limitation, and implementation relevance.
